1.1 A internet. 🔀 <==

A internet veio da arpanet, uma rede que foi criada em 1969 durante a guerra fria, foi criada pelos Estados Unidos, para guardar informações, caso as bases militares fossem atacadas.
Com a arpanet veio o primeiro protocolo criado, o NCP-Network control protocol, que faziam com que as bases se comunicassem mesmo tendo computadores diferentes de diferentes modelos, porém, com o crescimento da rede, percebeu-se que quando dois computadores estavam se comunicando, a rede parava e só podiam com que outros se comunicassem quando a outra comunicação estivesse terminada.
Para isso foi criado o TCP, com ele a rede não precisava parar quando dois computadores estivessem se comunicando, com ele veio também o IP que é como uma espécie de "cpf" dos computadores.

1.2 E como foi evoluindo? 🌐

Com isso, a arpanet foi crescendo, até que em 1972 já estava muito grande e inviável para os militares cuidarem, pois já haviam universidades e até empresas dentro da rede, com isso, a arpanet foi separada, a parte militar, científica e comercial, e dela veio a internet, com a mudança do nome em alguns tempos, como interconnectworking.

Hospedagem

Nós temos também os dominios e as hospedagens, DOMINIO é o nome dado ao seu site, ex: www.luan.com.br.
Hospedagem é o servidor onde você hospeda os arquivos do seu site, que serão entregues para os clientes na visita ao seu site.

Dominio

URL = Uniform resource locator, Localizador uniforme de recursos, nela temos alguns exemplos
www.luan.com.br = www é o sub-dominio, luan.com.br é o dominio, .com é o GTLD e .br é o CCTLD
luan.github.io, luan é o sub-dominio, github.io é o dominio , io é o GTLD, nela podemos ter um caminho também, exemplo: luan.github.io/exercicios/html, exercicios html é o caminho


1.3 O DNS E AS TLDS ®

Na internet, temos o DNS - Domain name system, que serve como se fosse uma "agenda" que guarda o nome dos sites, quando digitamos instagram.com por exemplo, o navegador pede para o dns procurar o ip do servidor com o nome instagram.com, nisso o DNS retorna o ip do servidor, dando acesso ao servidor do instagram e as coisas nele.

Modem

Também temos o modem que transforma as ondas de telefone, para ondas quadradas e as quadradas em de telefone, fazendo assim a: MODULAÇÃO E DEMODULAÇÃO dai vem a palavra modem

As TLDS, são os topos de dominio, nelas temos os GTLDS = Generic top level domain, e o CCTLD, Country code Top Level Domain os GTLDS mais conhecidos são: .com, .io, .info e alguns outros. os CCTLDS são ex: .br, .us, .uk, e etc.

Imagens dinâmicas e flexíveis.

O que são?

As pessoas usam diversos tipos de dispositivos, celular, tablets, pcs, televisores etc, para isso, precisamos que as imagens se adaptem a tela dos mesmos, uma tela de celular é menor do que a de um tablet ou pc por exemplo, a de uma televisão maior que de um computador e assim por diante, para isso temos as imagens dinâmicas, nós criamos algumas versões da foto que queremos em diferentes tamanhos, e nela usamos as tags de html para definir qual proporção de imagem aparecerá de acordo com a tela do dispositivo usado.

Tag para isso.

<source media="(max-width: px)" srcset="(fotoaqui)" type="image/png">

Para isso usamos a tag <picture>, junto dela usamos os atributos media, srcset e type, temos a tag inteira assim: <source> media="(max-width: (pixeis aqui)" srcset="(foto aqui)" type=image/png
Por ultimo antes de fechar a tag picture, colocamos a img src, importante ressaltar que: Os sources devem ser em ordem crescente, o menor em cima e os maiores embaixo.
Outra coisa é: Colocar uma largura um pouco acima da da imagem pode tornar mais interessante o resultado, exemplo: se a foto tem 1000px, colocar 1050px.

O que cada um significa?

type indica o media type da imagem que usamos, informações contidas no capitulo 10

srcset vai configurar o nome da imagem que será carregada quando o tamanho indicado for atingido.

media indica o tamanho maximo a ser considerado para carregar a imagem indicada no atributo srcset
